## 1. 着色器概念回顾

### 1.1 渲染管线流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    OpenGL ES 渲染管线                            │
└─────────────────────────────────────────────────────────────────┘

顶点数据 → 顶点着色器 → 图元装配 → 光栅化 → 片段着色器 → 像素输出
    │           │           │          │           │           │
    │           │           │          │           │           │
   坐标+颜色   位置处理    组成三角形  像素化     颜色计算    显示
```

### 1.2 着色器类型

| 着色器 | 处理对象 | 输入 | 输出 | 频率 |
|--------|---------|------|------|------|
| **顶点着色器** | 顶点 | attribute | gl_Position, varying | 每顶点一次 |
| **片段着色器** | 像素 | varying | gl_FragColor | 每像素一次 |

### 1.3 变量类型详解

```
┌─────────────────────────────────────────────────────────────────┐
│                    着色器变量类型                                │
└─────────────────────────────────────────────────────────────────┘

┌──────────────┬──────────────┬──────────────┬──────────────┐
│   uniform    │  attribute   │   varying    │   内置变量   │
├──────────────┼──────────────┼──────────────┼──────────────┤
│ 所有顶点共享 │ 每顶点不同   │ 顶点→片段传递 │ 系统定义     │
│ 矩阵、时间   │ 位置、颜色   │ 插值颜色     │ gl_Position  │
│ 外部传入     │ 缓冲区读取   │ GPU自动插值  │ gl_FragColor │
└──────────────┴──────────────┴──────────────┴──────────────┘

示例代码：
uniform mat4 u_Matrix;        // 投影矩阵（所有顶点用同一个）
attribute vec4 a_Position;    // 每个顶点的位置（各不相同）
attribute vec4 a_Color;       // 每个顶点的颜色（各不相同）
varying vec4 v_Color;         // 传递给片段着色器（自动插值）
```

### 1.4 插值原理

```
三角形颜色插值：

      顶点A（红色）
         │
        /│\
       / │ \
      /  │  \
     /   │   \
顶点B（绿色）──顶点C（蓝色）

片段着色器接收的 v_Color：
- 离 A 越近 → 红色权重越大
- 离 B 越近 → 绿色权重越大
- 离 C 越近 → 蓝色权重越大

计算公式：v_Color = wA × ColorA + wB × ColorB + wC × ColorC
         wA, wB, wC 是距离权重（和为 1）
```

## 2. 缓冲区操作回顾

### 2.1 为什么需要缓冲区？

```
┌─────────────────────────────────────────────────────────────────┐
│                    内存层次与性能                                │
└─────────────────────────────────────────────────────────────────┘

CPU 内存（JVM 堆）─────→ GPU 内存（显存）
        │                    │
        │  需要拷贝           │  直接访问
        │  性能低             │  性能高
        │                    │
        └────────────────────┘
              使用直接内存可避免拷贝
```

### 2.2 直接内存分配

```kotlin
// 创建直接内存缓冲区（Native Memory）
val buffer = ByteBuffer.allocateDirect(size * FLOAT_SIZE)
    .order(ByteOrder.nativeOrder())    // 设置本地字节序
    .asFloatBuffer()                   // 转换为 FloatBuffer

// 为什么用 nativeOrder？
// - CPU 和 GPU 字节序一致时效率最高
// - 避免字节序转换开销
```

### 2.3 数据格式：交错存储

```
┌─────────────────────────────────────────────────────────────────┐
│                    交错数据格式                                  │
└─────────────────────────────────────────────────────────────────┘

数据布局：[x, y, r, g, b, x, y, r, g, b, ...]
          └─────顶点0─────┘ └─────顶点1─────┘

优点：
- 一个缓冲区存储所有顶点数据
- 内存连续，访问效率高
- GPU 缓存友好

步幅（stride）计算：
stride = (坐标数 + 颜色数) × FLOAT_SIZE
       = (2 + 3) × 4 = 20 字节

设置属性指针：
// 位置属性：从 0 开始，跳过整个顶点数据
vertexBuffer.position(0)
glVertexAttribPointer(positionHandle, 2, GL_FLOAT, false, stride, vertexBuffer)

// 颜色属性：从 2 开始（跳过坐标），跳过整个顶点数据
vertexBuffer.position(2)
glVertexAttribPointer(colorHandle, 3, GL_FLOAT, false, stride, vertexBuffer)
```

### 2.4 索引缓冲（IBO）

```
┌─────────────────────────────────────────────────────────────────┐
│                    索引缓冲优势                                  │
└─────────────────────────────────────────────────────────────────┘

矩形绘制对比：

无索引（glDrawArrays）：
- 2 个三角形 × 3 个顶点 = 6 个顶点
- 数据：[x,y,r,g,b] × 6 = 30 个浮点数
- 有重复：顶点 1 和 2 被两个三角形共享

有索引（glDrawElements）：
- 4 个顶点 + 6 个索引
- 数据：[x,y,r,g,b] × 4 = 20 个浮点数 + 6 个 short
- 无重复：共享顶点通过索引复用

索引数据：
RECT_INDICES = shortArrayOf(0, 1, 2, 1, 3, 2)
                第一个三角形    第二个三角形
```

## 3. 矩阵变换回顾

### 3.1 正交投影矩阵

```
┌─────────────────────────────────────────────────────────────────┐
│                    正交投影原理                                  │
└─────────────────────────────────────────────────────────────────┘

作用：将世界坐标映射到 NDC [-1, 1]

可视范围定义：
orthoM(matrix, left, right, bottom, top, near, far)

屏幕适配策略：
┌─────────────┬─────────────┬─────────────┐
│   模式      │  固定轴     │  扩展轴     │
├─────────────┼─────────────┼─────────────┤
│ 横屏(>1)    │ 上下 ±1    │ 左右 ±ratio │
│ 竖屏(<1)    │ 左右 ±1    │ 上下 ±ratio │
└─────────────┴─────────────┴─────────────┘

示例：16:9 屏幕（横屏）
aspectRatio = 16/9 = 1.78
orthoM(projectionMatrix, -1.78, 1.78, -1, 1, -1, 1)

结果：
- 可视范围：x[-1.78,1.78], y[-1,1]
- 矩形(-100,100) 映射后 ≈ [-0.056, 0.056]（占屏幕一小部分）
```

### 3.2 平移矩阵

```
┌─────────────────────────────────────────────────────────────────┐
│                    平移矩阵推导                                  │
└─────────────────────────────────────────────────────────────────┘

目标：将点 (x, y) 移动 (tx, ty)

齐次坐标：(x, y, 1) → 用 3D 表示 2D 点

平移矩阵：
┌                    ┐
│  1    0    tx     │
│  0    1    ty     │
│  0    0    1      │
└                    ┘

计算：
[x']   [x + tx]
[y'] = [y + ty]
[1 ]   [1     ]

API 使用：
Matrix.setIdentityM(modelMatrix, 0)    // 重置为单位矩阵
Matrix.translateM(modelMatrix, 0, tx, ty, 0)  // 应用平移
```

### 3.3 动画实现

```
┌─────────────────────────────────────────────────────────────────┐
│                    时间驱动的动画                                │
└─────────────────────────────────────────────────────────────────┘

核心代码：
val elapsedSeconds = (currentTime - startTime) / 1000f
val tx = sin(elapsedSeconds * frequency) * amplitude

三角函数特性：
- sin(x): 范围 [-1, 1]，周期 2π
- cos(x): 范围 [-1, 1]，周期 2π
- 乘以 amplitude 放大范围
- 乘以 frequency 改变速度

动画类型：
┌──────────────┬──────────────────────────────┐
│   效果       │   公式                       │
├──────────────┼──────────────────────────────┤
│ 左右摆动     │ tx = sin(t) × amplitude      │
│ 上下摆动     │ ty = cos(t) × amplitude      │
│ 圆周运动     │ tx = cos(t), ty = sin(t)     │
│ 李萨如曲线   │ tx = sin(t×f1), ty=cos(t×f2) │
└──────────────┴──────────────────────────────┘
```

### 3.4 矩阵乘法顺序

```
┌─────────────────────────────────────────────────────────────────┐
│                    变换顺序很重要                                │
└─────────────────────────────────────────────────────────────────┘

顶点变换公式：
gl_Position = Projection × Model × Vertex

矩阵乘法从右到左执行：
1. Model 变换：先对物体本身操作
2. Projection 变换：最后投影到屏幕

组合矩阵计算：
Matrix.multiplyMM(resultMatrix, 0, projectionMatrix, 0, modelMatrix, 0)

传递给着色器：
glUniformMatrix4fv(matrixHandle, 1, false, resultMatrix, 0)
```

## 4. 完整渲染流程总结

```
┌─────────────────────────────────────────────────────────────────┐
│                    OpenGL ES 渲染完整流程                        │
└─────────────────────────────────────────────────────────────────┘

onSurfaceCreated（初始化）：
  1. glClearColor()          ─ 设置清屏颜色
  2. 编译着色器              ─ loadShader()
  3. 创建程序                ─ glCreateProgram()
  4. 链接程序                ─ glLinkProgram()
  5. 获取句柄                ─ glGetAttribLocation/glGetUniformLocation
  6. 创建缓冲区              ─ ByteBuffer.allocateDirect()

onSurfaceChanged（屏幕适配）：
  1. glViewport()            ─ 设置视口
  2. 计算宽高比              ─ aspectRatio = width / height
  3. 创建投影矩阵            ─ Matrix.orthoM()

onDrawFrame（每帧绘制）：
  1. glClear()               ─ 清除屏幕
  2. 计算动画时间            ─ elapsedSeconds
  3. 更新模型矩阵            ─ translateM/rotateM/scaleM
  4. 组合矩阵                ─ multiplyMM()
  5. glUseProgram()          ─ 激活程序
  6. glUniformMatrix4fv()    ─ 传递矩阵
  7. 设置顶点属性            ─ glVertexAttribPointer()
  8. 绘制                    ─ glDrawElements()
  9. 清理                    ─ glDisableVertexAttribArray()
```
